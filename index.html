SabaiBill

ระบบจัดการเอกสาร SME

SabaiBill คือแอปจัดการเอกสารซื้อขายสำหรับ SME — รวมการสร้าง แปลงต่อเนื่อง และพิมพ์/บันทึก PDF ไว้ที่เดียว เน้นใช้งานจริงและนำเสนอ demo ได้ทันที

โปรเจกต์นี้ช่วยให้ผู้ใช้

  • สร้างและจัดเก็บเอกสาร QT (ใบเสนอราคา), SO (ใบสั่งขาย), DO (ใบส่งของ), IV (ใบแจ้งหนี้)
  • แปลงต่อเนื่อง เช่น QT → SO → DO / IV โดยไม่ต้องกรอกซ้ำทั้งฉบับ
  • ตั้งค่าข้อมูลบริษัท ธนาคาร เงื่อนไขท้ายบิล เทมเพลต ลายเซ็น QR ชำระเงิน และสกุลเงิน
  • คำนวณ VAT ส่วนลด หัก ณ ที่จ่าย (WHT) และแสดงยอดสุทธิ
  • พิมพ์ / บันทึก PDF จากหน้า Preview
  • จัดการลูกค้าและสินค้า พร้อมค้นหาแบบ autocomplete

การเข้าใช้งาน: ไม่มีหน้าสมัครหรือล็อกอินแยก — กดปุ่มเดียวแล้วระบบใช้ผู้ใช้คนเดียวในฐานข้อมูล เหมาะกับ demo / ทดลองภายใน ไม่แนะนำสำหรับข้อมูลลับหากไม่มีชั้นป้องกันอื่น

ภาษาและประสบการณ์ผู้ใช้

  • ✦ ภาษาอินเทอร์เฟซ: ไทยและอังกฤษ (สลับได้ในแอป)
  • ✦ ข้อความในเอกสาร: รองรับฟิลด์คู่ไทย/อังกฤษ (ชื่อบริษัท ที่อยู่ ลูกค้า ฯลฯ)
  • ✦ ออกแบบให้อ่านง่าย: ตัวอักษรโต ปุ่มกดใหญ่ เหมาะผู้ใช้ที่ต้องการความชัดเจน

บทเรียนจาก MINI-ERP & ความท้าทาย

SabaiBill = ย่อสเกลลงมา เน้นใช้ง่าย — แต่ยังมีจุดต้องเก็บ

  • ทิศทางโปรเจกต์: ตั้งใจให้เป็นคำตอบจากประสบการณ์ MINI-ERP — โฟกัสแค่เอกสารซื้อขาย ลดความซับซ้อนของระบบรอบข้าง
  • Flow ยังไม่แน่นอน: ลำดับ QT → SO → DO / IV บางจุดยังต้องอธิบายผู้ใช้ — story การใช้งานยังไม่นิ่ง อยากให้ “รู้ว่าขั้นถัดไปคืออะไร” โดยไม่ต้องจำ
  • รายละเอียดบิล: โครงหน้าตาแต่ละประเภทยังรู้สึกใกล้เคียงกัน — SME แต่ละเจ้าต้องการความยืดหยุ่น (ฟิลด์ / layout / เงื่อนไข) มากกว่านี้
  • โค้ด (สั้นๆ): การ map ฟิลด์ระหว่างเอกสารกับ state ใน React บางส่วนยังผูกตาย ทำให้ขยายเทมเพลตหรือ validation ซ้ำ — กำลังมอง abstraction และ schema ที่ชัดขึ้น

ออกแบบเวิร์กโฟลว์บน Make.com (ชุด Integration LINE): รับเหตุการณ์จาก LINE → ดาวน์โหลดไฟล์แนบรูป → ส่งรูปไป OCR ผ่านโมดูล HTTP → ใช้ Text parser ดึงเลขพัสดุตามรูปแบบที่กำหนด → บันทึกแถวใน Google Sheets → ส่งข้อความแจ้งสถานะที่เตรียมไว้กลับลูกค้าด้วย LINE Push Message โดยตั้งรันแบบ ทันทีเมื่อมีข้อมูลเข้า

ที่มา & ข้อจำกัดจริง

อัตโนมัติครั้งแรกที่ผสม AI กับงานประจำวัน — ยังต้องพัฒนาต่อ

  • แรงบันดาลใจ: อยากส่งเลขพัสดุให้ทีมเร็ว ไม่ต้องนั่งพิมพ์ซ้ำ — ใช้ AI ช่วยคิด flow และต่อกับ Make.com เป็นครั้งแรก
  • OCR กับภาษาไทย: บริการที่ใช้อ่านตัวอักษรไทยบนสลิปจริงยังไม่นิ่ง ฟอนต์เล็ก/เบลอทำให้ต้องพึ่ง regex และการตรวจซ้ำ
  • ทีละรูป & หลายข้อความ: การแคปหรือแนบหลายใบพร้อมกันทำให้สับสนได้ — ต้องจัดลำดับ scenario และ error handling ใน Make ให้ชัด
  • ทิศทางต่อไป: ลอง engine OCR ที่เหมาะกับไทยมากขึ้น รองรับ batch หรือแจ้งผู้ใช้เมื่อความมั่นใจของผลอ่านต่ำ

  • ฟีเจอร์เด่น: ระบบจัดซื้อ (PR → PO → GRN พร้อมอัปเดตสต็อก), ระบบขาย (Quotation → SO → DO และลิงก์ยืนยันรับของ), ระบบ Master Data (สินค้า, คู่ค้า, คลัง, หน่วยนับ) และการจัดการสิทธิ์
  • เทคโนโลยี (Tech Stack): Backend: Python, Django 6, DRF, JWT, PostgreSQL, Gunicorn | Frontend: Vue 3, Vite, JavaScript, Tailwind CSS
