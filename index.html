MINI-ERP System

Smart ERP — Web-based business system

A web-based ERP for small-to-medium businesses. Brings purchasing, receiving, inventory, sales, delivery, basic finance, and document approvals into one place. Clean separated-API architecture, modern UI, secure JWT login — runs smoothly on desktop and mobile.

Core modules
  • 📦

  • 🛒

  • 🗂️

SabaiBill

SME Document Management System

SabaiBill is a sales-doc app for SMEs — create, chain, and print/save PDFs in one place. Built to be real-world useful and instantly demoable.

What users can do
  • Create and store QT (quotation), SO (sales order), DO (delivery order), IV (invoice).
  • Chain documents — QT → SO → DO/IV — without re-entering data.
  • Configure company info, bank details, terms, templates, signatures, payment QR, and currency.
  • Auto-calculate VAT, discounts, withholding tax (WHT), and net totals.
  • Print / save PDF directly from the preview page.
  • Manage customers and products with autocomplete search.

Access: no separate sign-up / login — one-button entry using a shared DB user. Built for demo / internal trial; not for sensitive data without an extra security layer.

Language & UX
  • ✦
    Interface in TH and EN (toggle in-app).
  • ✦
    Document fields support bilingual TH/EN (company, address, customer, etc.).
  • ✦
    Readable typography, large buttons — designed for clarity.
Lessons from MINI-ERP & what's next
  • Direction: a deliberate step down from MINI-ERP — focus only on trade documents, less surrounding system noise.
  • Flow uncertainty: QT → SO → DO/IV still needs better guidance — users should "know the next step" without remembering.
  • Bill detail: doc types feel too similar — SMEs need more flexible fields / layouts / conditions.
  • Code (briefly): field mapping between doc and React state is partly hard-coded — exploring cleaner abstractions and a schema layer.

Workflow Automation

Parcel Tracking (Make.com)

A Make.com workflow (Integration LINE): receive a LINE event → download the attached image → send to OCR via HTTP → parse the tracking number with the Text parser → log a row in Google Sheets → push a status message back to the customer via LINE — running immediately as data arrives.

Removes the admin's repetitive typing. The flow shrinks to Take photo → Wait → Copy → Send, while OCR, sheet logging, and notifications all happen automatically.
